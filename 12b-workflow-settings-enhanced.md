# Phase 12B: Enhanced Workflow Settings & Entity Attribute Mapping

## Overview

This document extends Phase 12 with enhanced UI mockups for workflow configuration, focusing on:
1. **Workflow slot configuration** - Upload and configure ComfyUI workflows for each asset type
2. **Entity attribute mapping** - Define which character/location attributes auto-populate generation prompts
3. **Generation modal integration** - Dynamic forms based on workflow configuration

---

## User Stories

### Epic: Workflow Slot Management

#### US-12B.1: View All Workflow Slots
**As a** DM
**I want to** see all available workflow slots grouped by entity type
**So that** I can configure generation workflows for different asset types

**Acceptance Criteria:**
- Settings page shows workflow slots grouped: Characters, Locations, Items, Maps
- Each slot shows: name, dimensions, configured status, workflow name if configured
- Visual indicators: âœ“ green for configured, â—‹ gray for unconfigured
- "Configure" button on each slot to upload/paste workflow

---

#### US-12B.2: Upload Workflow JSON
**As a** DM
**I want to** upload a ComfyUI workflow JSON file (API format)
**So that** I can use custom workflows for asset generation

**Acceptance Criteria:**
- Modal with tabs: "Upload File" | "Paste JSON"
- File picker accepts `.json` files
- Paste area with syntax highlighting
- "Paste from Clipboard" button
- Real-time JSON validation with error display
- Preview of detected nodes/inputs before saving

---

#### US-12B.3: Configure Prompt Mapping
**As a** DM
**I want to** specify which workflow inputs receive the generation prompt
**So that** my text prompts go to the correct nodes

**Acceptance Criteria:**
- List all text inputs from workflow
- Mark inputs as: "Primary Prompt", "Negative Prompt", or "Don't Map"
- Visual preview of current value in each input
- At least one Primary Prompt required to save

---

#### US-12B.4: Set Default Parameter Values
**As a** DM
**I want to** configure default values for workflow parameters
**So that** I don't have to set them every time I generate

**Acceptance Criteria:**
- Form showing all non-connection inputs grouped by node
- Appropriate input types: text, number, dropdown, checkbox
- "Lock" checkbox per input (locked = hidden from generation modal)
- "Reset to Original" to restore workflow default
- Save defaults persistently

---

### Epic: Entity Attribute Mapping

#### US-12B.5: Configure Entity Attribute Mapping
**As a** DM
**I want to** define which entity attributes auto-populate the prompt
**So that** character/location details are automatically included

**Acceptance Criteria:**
- Per-slot configuration for attribute mapping
- Character slots: select from name, archetype, description, wants, fears, backstory
- Location slots: select from name, type, description, atmosphere, features
- Drag-and-drop ordering of attributes
- Template preview showing how prompt will be constructed

---

#### US-12B.6: Prompt Template Editor
**As a** DM
**I want to** customize the prompt template for each slot
**So that** I can control how attributes are combined

**Acceptance Criteria:**
- Template string with placeholders: `{name}`, `{archetype}`, `{description}`
- Default template: `"A portrait of {name}, {archetype}. {description}"`
- Live preview with sample entity data
- Option to use plain text with mapped attributes appended

---

### Epic: Generation Modal Integration

#### US-12B.7: Dynamic Generation Form
**As a** DM
**I want to** see workflow-specific options in the generation modal
**So that** I can adjust parameters before generating

**Acceptance Criteria:**
- Prompt field pre-populated from entity attributes
- Editable - user can modify before generating
- "Advanced Options" section with unlocked parameters
- Seed control: Random, Specific number, Reuse last
- Variations count selector

---

## Data Model Extensions

### EntityAttributeMapping

```rust
/// Mapping of entity attributes to prompt generation
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct EntityAttributeMapping {
    /// Ordered list of attributes to include
    pub attributes: Vec<EntityAttribute>,
    /// Prompt template with placeholders
    pub template: String,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum EntityAttribute {
    // Character attributes
    CharacterName,
    CharacterArchetype,
    CharacterDescription,
    CharacterWants,
    CharacterFears,
    CharacterBackstory,
    // Location attributes
    LocationName,
    LocationType,
    LocationDescription,
    LocationAtmosphere,
    LocationFeatures,
    LocationSecrets,
    // Item attributes
    ItemName,
    ItemType,
    ItemDescription,
    ItemProperties,
}

impl Default for EntityAttributeMapping {
    fn default() -> Self {
        Self {
            attributes: vec![
                EntityAttribute::CharacterName,
                EntityAttribute::CharacterArchetype,
                EntityAttribute::CharacterDescription,
            ],
            template: "A portrait of {name}, a {archetype}. {description}".to_string(),
        }
    }
}
```
